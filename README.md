# Project 01: Fundamentals

## Purpose

Programming is not just about writing code that runs. It is also about using code to represent ideas, solve problems, and communicate information clearly.

In this project, you will create a Java application connected to a real-world topic, community issue, personal interest, or practical idea. Your program does not need to be large or complicated. The goal is to show that you can use the programming tools we have practiced so far in a meaningful way.

## Learning Objectives

By completing this project, you will practice:

- Creating and using variables
- Working with `String`, `int`, and `double` values
- Using operators to calculate or transform data
- Using `if` statements to make decisions
- Defining and calling static methods
- Organizing your code so that it is readable and understandable

## Project Overview

Create a Java program that connects to something outside of the textbook.

You may choose a topic inspired by:

- an article
- a podcast
- a video
- a community issue
- a personal interest
- a useful or fun idea
- an aspect of your final project

Your program should demonstrate the concepts we have covered so far: data types, variables, operators, `if` statements, and static methods.

Choose a topic that gives your program a clear purpose. For example, your program might estimate a cost, compare two options, classify a result, calculate a score, convert information, or help someone make a simple decision.

## Completion Specifications

To earn a ✅ complete on this project, your submission must satisfy **all** of the following specifications:

- Your program connects to a real-world topic, community issue, personal interest, or practical idea.
- Your program uses only Java concepts we have covered so far:
  - data types
  - variables
  - operators
  - `if` statements
  - static methods
- Your program includes at least 3 variables.
- Your program uses each of the following data types at least once:
  - `String`
  - `int`
  - `double`
- Your program defines and uses at least 2 distinct static methods.
- Each static method is used in a meaningful way.
- Your program demonstrates a calculation or transformation of data.
- Your code follows the coding standards listed below.

## Going Further

In addition to the Completion Specifications, completing this section **exceeds minimum specifications**.

To have this project count as exceeding minimum specifications, your submission must satisfy **all** of the following specifications:

- Your class is documented using JavaDoc.
- Each method is documented using JavaDoc.
- The class-level JavaDoc includes a clear description of what the program does.
- The class-level JavaDoc includes attribution using the `@author` tag.
- Each method’s JavaDoc explains the purpose of the method.
- Any method with parameters uses `@param` tags.
- Any method with a return value uses an `@return` tag.

## Coding Standards

All submissions should:

- Use meaningful identifiers.
- Follow Java naming conventions.
- Remove auto-generated comments and unused code.
- Use comments only when they improve clarity.
- Format code cleanly and consistently.
- Use proper indentation and brace alignment.
- Use whitespace to improve readability.
- Make prompts and output clear, readable, and professional.

Readable code matters. Someone else should be able to look at your program and understand the purpose of the program, the role of each method, and the meaning of your output.
